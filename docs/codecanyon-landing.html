Install

Three paths. Pick the one that matches your hosting.

All three end with the install wizard at /admin.

1

Docker (recommended)

~5 minutes. Bundled compose brings up the whole stack plus Caddy auto-TLS.

# Extract + configure
unzip restaurant-pos-v1.0.0.zip
cd restaurant-pos-v1.0.0
cp .env.example .env
nano .env  # DATABASE_URL, JWT

# Start
docker compose up -d --build

# Visit
https://yourdomain.com/admin
2

Ubuntu VPS (no Docker)

Full control. Node 22 + Postgres 15 + PM2 + nginx + certbot.

# Node 22 from NodeSource
curl -fsSL https://deb.nodesource.com/setup_22.x \
  | sudo -E bash -
sudo apt install -y nodejs unzip
sudo npm i -g pnpm pm2

unzip restaurant-pos-v1.0.0.zip
cd restaurant-pos-v1.0.0
pnpm install --frozen-lockfile --prod
pnpm prisma migrate deploy
pm2 start api/dist/main.js --name pos
3

cPanel shared hosting

Node.js App Manager + Postgres from cPanel's database tools. No SSH required.

1. Create PG database in cPanel
2. File Manager → upload zip → extract
3. Setup Node.js App:
   Node: 22
   Root: ~/restaurant-pos
   Startup: api/dist/main.js
4. Run NPM Install (in cPanel UI)
5. Terminal:
   pnpm prisma migrate deploy
   pnpm db:seed:empty

FAQ

Questions buyers usually have.

Short versions below. Full details in the included docs/FAQ.md inside the zip.

Do I need an internet connection?

Only for the daily license check. If you go offline, you have a 7-day grace period during which everything keeps working. After 7 days offline, mutations are blocked but reads (reports, history) stay open. Activation is the only step that strictly requires internet.

Can I run this on shared cPanel hosting?

Yes, if your host supports Node.js 22 and PostgreSQL 15. Most modern cPanel installs include both via the Node.js App Manager feature. See install path #3.

How do customisations survive updates?

Database content (menu, customers, orders, settings) survives everything. Source modifications get overwritten by the buyer-zip update path. If you're modifying source, fork the repo and merge upstream releases manually rather than using the in-app updater.

Does the license check phone home with my data?

No. Only your purchase code, your domain, and an opaque per-machine fingerprint. Source for the verification client is in the zip — every line is auditable.

What happens if your license server goes down?

Your install keeps working for up to 7 days on the locally-cached signed proof. The proof is ed25519-signed with a key bundled at build time, so even if our server is unreachable you operate normally.

Can I move my install to a new domain?

Yes. Settings → License → Deactivate (releases the seat), then re-activate on the new domain. Each license can only be on one domain at a time. Wildcard licenses (*.yourchain.com) cover any subdomain depth — buy one of those if you run multiple branded subdomains for franchises.

Do I get the source code?

Yes — the full TypeScript source for API, admin, POS, KDS, QR-order, and public website. Not minified, not obfuscated. Modify freely under the CodeCanyon Regular or Extended terms.

Can I run this multi-tenant for several restaurants?

Each install is single-tenant. For SaaS with multiple independent restaurants, use an Extended license per tenant. Most buyers use the built-in multi-branch feature instead — one install can support many physical locations under the same owner.

How do I get support?

CodeCanyon item comments for quick questions; the support link in the item description for tickets. Custom development isn't included with the regular license but is available on request.
