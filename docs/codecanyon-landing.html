Multi-branch

One install, many locations. Branch-scoped data with shared owners + reports.

Install guide

Three paths. All three end with the install wizard at /admin.

1) Docker Compose (easiest, ~5 minutes)

Best when your VPS already has Docker installed. The bundled compose file brings up the API, all SPAs, Postgres, and Caddy.

tar -xzf restaurant-pos-v1.0.0.zip
cd restaurant-pos-v1.0.0
cp .env.example .env
nano .env            # fill DATABASE_URL, JWT_SECRET, etc
docker compose up -d
# visit https://yourdomain.com/admin → run the install wizard
2) Ubuntu VPS (no Docker)

Manual setup with PM2 + nginx. Good when you want full control.

# Node 22 + Postgres 15 + PM2
sudo apt install nodejs postgresql nginx
sudo npm install -g pm2 pnpm

# Install + build
tar -xzf restaurant-pos-v1.0.0.zip
cd restaurant-pos-v1.0.0
pnpm install --frozen-lockfile
cp .env.example .env
nano .env
pnpm prisma migrate deploy
pnpm db:seed:empty   # or db:seed:demo-light
pnpm exec turbo run build

# Start API + serve SPA dists via nginx
pm2 start apps/api/dist/main.js --name pos-api
sudo cp infra/nginx-example.conf /etc/nginx/sites-enabled/pos
sudo systemctl reload nginx
3) cPanel (shared hosting with Node.js App Manager)

Works on most cPanel hosts that ship Node 22. Uses your hosting provider's Postgres + nginx. See docs/INSTALL.md for the full step-by-step.

  • Create a Node.js app (Node 22, document root → apps/api/dist).
  • Add a Postgres DB from cPanel's database tools.
  • Upload the zip via File Manager, extract.
  • SSH in once to pnpm install and pnpm prisma migrate deploy.
  • cPanel's Node App Manager handles starting the API; nginx serves the SPAs.

Updates

Two paths. Pick whichever fits your hosting.

  • Admin UI re-upload (recommended). Download the new release zip, upload it via Settings → Updates. The system takes a backup, applies migrations, swaps the running version, and rolls back automatically if the post-update health check fails.
  • SSH update. Re-extract the zip over the install dir, run pnpm install + pnpm prisma migrate deploy, restart with pm2 reload pos-api.

Either way: your custom branding, menu, customers, and order history stay put. Only the application code changes.

FAQ

Do I need an internet connection?

The license verification requires a quick internet check at activation and once per day to keep your install in "active" mode. If you go offline, you have a 7-day grace period during which everything keeps working. After 7 days offline, mutations are blocked but reads (reports, history) stay open.

Can I run this on shared cPanel hosting?

Yes, as long as your host supports Node.js 22 and Postgres 15. Most modern cPanel installs include both via the "Node.js App Manager" feature.

How do customisations survive updates?

Branding, menu, prices, staff, customers, and reports all live in the database — updates only touch application code. Theme overrides go in apps/admin/src/styles/branding.css which is preserved during admin-UI updates. For deeper code customisation, fork the repo and merge upstream changes manually.

Does the license check phone home with my data?

No. The license check sends only your purchase code, your domain, and an opaque machine fingerprint. Nothing about your menu, orders, or customers leaves your server. Source for the verification client ships with the product — you can read every line.

What happens if your license server goes down?

Your install keeps working for up to 7 days on its cached license proof. The proof is signed with a key bundled at build time, so even if our server is unreachable you can still operate normally.

Can I move my install to a new domain?

Yes. From the admin UI, deactivate the license (releases the seat), then activate again on the new domain. Each license can only be on one domain at a time.

Do I get the source code?

Yes — the full TypeScript source for the API, admin, POS, KDS, QR-order PWA, and public website. Not minified, not obfuscated. Modify freely under the CodeCanyon Regular or Extended license terms (see LICENSE.txt).

Is the POS Desktop edition included?

No — it's a separate purchase. Buy the desktop edition if you want the Windows app for cashier terminals. The web edition's POS works fine in any browser, including a wall-mounted touch screen running Chrome in kiosk mode.

How do I get support?

Use the CodeCanyon item's comment thread for general questions, or open a support ticket via the link in the item description. Custom development isn't included with the regular license.

Can I run this multi-tenant for several restaurants?

Each install is single-tenant by design. To run a SaaS service yourself with this, you'd need an Extended license per tenant. Most buyers use the multi-branch feature instead — one install can support many physical locations under the same owner.
